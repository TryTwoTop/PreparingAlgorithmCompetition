2. Boring Numbers (7pts, 12pts)
지루한 숫자

문제
Ron은 지루한 숫자에 대해 책을 읽는다. 

//
책에 따르면, 숫자의 짝수 자리에 있는 모든 숫자가 짝수이고 홀수 위치에 있는 모든 숫자가 홀수이면 양수를 지루하다고 합니다. 숫자는 1부터 시작하여 왼쪽에서 오른쪽으로 열거됩니다. 예를 들어, 홀수 위치에는 홀수인 {1, 7}이 포함되고 짝수 위치에는 짝수인 {4, 8}이 포함되므로 숫자 1478은 지루합니다.

두 개의 숫자 L 및 R이 주어지면 Ron은 [ L , R ] ( L 및 R 포함) 범위에서 얼마나 많은 숫자가 지루한지 계산하려고 합니다. Ron은 문제를 해결할 수 없으므로 귀하의 도움이 필요합니다.

입력
입력의 첫 번째 줄은 테스트 케이스의 수 T를 제공 합니다. T 테스트 케이스는 다음과 같습니다. 각 테스트 케이스는 두 개의 숫자 L 및 R 이있는 단일 행으로 구성됩니다 .

출력
각 테스트 케이스에 대해를 포함하는 한 줄을 출력합니다 Case #x: y. 여기서는 x테스트 케이스 번호 (1부터 시작)이고 y는 지루한 숫자의 개수입니다.
//

범위
시간 제한: 20초
메모리 제한: 1기가
1 <= T <= 100.

테스트 세트 1
1 <= L <= R <= 10^3.

테스트 세트 2
1 <= L <= R <= 10^18.

샘플
입력		출력
3 15		Case #1: 6
120 125		Case #2: 3
779 783		Case #3: 2
